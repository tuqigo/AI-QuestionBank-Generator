# 题目模板配置需求实现步骤

本文档总结了一类常见需求的实现步骤：**配置或更新题目模板记录，使其能够按照指定要求生成题目**。

典型需求示例：
> "查询 question_templates 的 id 为 14 的记录，理解其题目要求（年级、知识点、name、description），填充 template_pattern、variables_config 和 generator_module 字段，使模板能正常生成题目。"

---

## 工作流程（测试驱动）

**必须采用测试驱动开发（TDD）模式**：

```
1. 编写测试用例（定义预期行为）
2. 运行测试 → 失败（RED）
3. 实现/修改代码使测试通过（GREEN）
4. 重构代码并确保持续通过（REFACTOR）
```

**测试文件位置**：`backend/tests/template_config_test.py`

**测试覆盖要求**：
- 每个模板配置必须有对应的测试用例
- 测试必须验证数字范围、运算类型、结果约束
- 测试必须能捕获超纲题目（如 5 以内生成 100 以内）

## 实现步骤

### 步骤 0：编写测试（TDD 第一步）

**在 `backend/tests/` 目录下创建测试文件**：

```python
# tests/template_config_test.py
import unittest
import json
import sys
sys.path.insert(0, '..')

from utils.db_helper import DbHelper
from services.template.generators.mixed_addition_subtraction import MixedAdditionSubtractionGenerator


class TestTemplateConfiguration(unittest.TestCase):
    """模板配置测试类"""

    def setUp(self):
        self.generator = MixedAdditionSubtractionGenerator()

    def test_id_18_5 以内数_结果不超过 5(self):
        """ID=18: 5 以内数的认识，结果不能超过 5"""
        template = DbHelper.get_template(18)
        config = json.loads(template['variables_config'])
        questions = self.generator.generate(config, 50, 'ORAL_CALCULATION')

        for q in questions:
            stem = q['stem']
            # 提取结果验证
            if '=' in stem:
                result = int(stem.split('=')[0].split('[BLANK]')[0])
                self.assertLessEqual(result, 5, f"题目 {stem} 结果超过 5")

    def test_id_27_比大小带运算_10 以内 (self):
        """ID=27: 比大小（带运算），结果不超过 10"""
        template = DbHelper.get_template(27)
        config = json.loads(template['variables_config'])
        questions = self.generator.generate(config, 50, 'ORAL_CALCULATION')

        for q in questions:
            stem = q['stem']
            # 验证是单步运算，不是三步混合
            op_count = stem.count('+') + stem.count('-')
            self.assertLessEqual(op_count, 2, f"题目 {stem} 不是单步运算")


if __name__ == '__main__':
    unittest.main()
```

### 步骤 1：查询并理解模板记录

使用 DbHelper 查询模板记录，理解其要求：

```python
from utils.db_helper import DbHelper

# 查询模板记录
template = DbHelper.get_template(14)
print(f"name: {template['name']}")
print(f"grade: {template['grade']}")
print(f"subject: {template['subject']}")
print(f"question_type: {template['question_type']}")
print(f"example: {template['example']}")
print(f"knowledge_point_id: {template['knowledge_point_id']}")
print(f"description: {template['description']}")

# 查询关联的知识点
if template['knowledge_point_id']:
    kp = DbHelper.get_knowledge_point(template['knowledge_point_id'])
    print(f"知识点：{kp['name']}")
```

**需要理解的要素**：
- **年级** (`grade`)：如 `grade2` 表示二年级
- **学科** (`subject`)：如 `math` 表示数学
- **题目类型** (`question_type`)：如 `ORAL_CALCULATION` 表示口算题
- **示例题目** (`example`)：如 `["26+12"]` 提示题目格式
- **知识点** (`knowledge_point_id` → `knowledge_points.name`)：如 "100 以内的加法和减法（二）"
- **模板名称和描述** (`name`, `description`)：如 "两位数加两位数（不进位）"

**重要：多维度综合考虑**：
配置模板时，必须同时考虑以上所有要素，特别是**知识点名称**和**示例题目**，它们决定了数字范围和运算类型。

例如：
- 知识点 "5 以内数的认识" → 数字范围 0-5，结果不超过 5
- 知识点 "6~10 的认识" → 数字范围 6-10，或结果不超过 10
- 知识点 "100 以内的加法和减法" → 数字范围和结果可达 100
- 示例 "2+4○7" → 单步运算后比较，数字较小
- 示例 "74-28+22○75" → 两步混合运算后比较

**错误示例**（已发生的教训）：
- 知识点是 "6~10 的认识"，但生成了 29-49+31 等 100 以内数字 → 错误：没有考虑知识点范围
- 知识点是 "5 以内数的认识"，但生成了 5+5=10 → 错误：结果超出 5，需要添加 `result_within: 5`
- 比大小（带运算）生成了三步混合运算 → 错误：示例 "2+4○7" 是单步运算

### 步骤 3：扩展生成器（如需要）

如果现有生成器不支持所需约束，需要扩展它。

**示例：为加减法生成器添加 `ensure_no_carrying` 约束**

编辑 `backend/services/template/generators/mixed_addition_subtraction.py`：

```python
# 1. 在 generate() 方法中读取新配置参数
ensure_no_carrying = template_config.get("ensure_no_carrying", False)  # 加法不进位

# 2. 在 simple 题型中添加约束检查
if q_type == "simple":
    a = random.randint(num_min, num_max)
    b = random.randint(num_min, num_max)
    op = random.choice(op_values)

    # 检查不进位约束（仅加法）
    if ensure_no_carrying and op == "+":
        # 个位相加不能进位
        if (a % 10) + (b % 10) >= 10:
            continue
        # 十位相加不能进位
        if (a // 10) + (b // 10) >= 10:
            continue

    result = a + b if op == "+" else a - b
    stem = f"{a}{op}{b}=[BLANK]"
```

**如需添加新的知识点**，更新 `get_knowledge_points()` 方法：

```python
def get_knowledge_points(self, template_config: dict) -> List[str]:
    points = ["百以内加减法"]
    if template_config.get("ensure_no_carrying", False):
        points.append("两位数加两位数（不进位）")
    return points
```

### 步骤 4：确定要填充的字段

需要填充的字段：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `template_pattern` | 模板模式字符串 | `{a} + {b} = [BLANK]` |
| `variables_config` | JSON 配置 | 见下方 |
| `generator_module` | 生成器模块名 | `mixed_addition_subtraction` |
| `is_active` | 是否启用 | `1` |

**variables_config 配置示例**（两位数加两位数不进位）：

```json
{
  "num": {"min": 10, "max": 99},
  "op": {"values": ["+"]},
  "question_complexity": ["simple"],
  "ensure_no_carrying": true,
  "ensure_sum_within_100": true,
  "rendering_config": {
    "layout": "multi",
    "columns": 4,
    "font_size": 18,
    "answer_width": 32,
    "answer_style": "blank",
    "show_question_number": true
  }
}
```

**variables_config 配置示例**（5 以内数的认识）：

```json
{
  "num": {"min": 0, "max": 5},
  "op": {"values": ["+", "-"]},
  "question_complexity": ["simple"],
  "rules": ["ensure_positive"],
  "result_within": 5,
  "rendering_config": {
    "layout": "multi",
    "columns": 5,
    "font_size": 18,
    "answer_width": 32,
    "answer_style": "blank",
    "show_question_number": true
  }
}
```

**variables_config 配置示例**（比大小带运算，结果≤10）：

```json
{
  "num": {"min": 1, "max": 10},
  "op": {"values": ["+", "-"]},
  "question_complexity": ["compare_with_result"],
  "rules": ["ensure_positive", "result_within_10"],
  "q_type": {"compare_with_result": "circle"},
  "rendering_config": {
    "layout": "multi",
    "columns": 4,
    "font_size": 18,
    "answer_width": 32,
    "answer_style": "circle",
    "show_question_number": true
  }
}
```

**关键配置说明**：
- `num`: 数字范围，根据知识点决定（如 "5 以内" 则 0-5，"6~10 的认识" 则根据题型可能是 1-10）
- `result_within`: 结果上限，用于限制加减法的结果（如 5 以内数的认识，结果不能超过 5）
- `rules`: 规则列表，常用规则：
  - `ensure_positive`: 确保结果非负（减法时被减数≥减数）
  - `ensure_different`: 确保两个数字不同（比大小时避免相等）
  - `result_within_10`: 结果不超过 10（用于 10 以内运算）
  - `result_within_100`: 结果不超过 100
- `question_complexity`: 题型复杂度，可选值：
  - `simple`: 简单加减法
  - `compare_simple`: 简单比大小
  - `compare_with_result`: 先运算后比较（单步运算）
  - `consecutive_add`: 连加
  - `consecutive_subtract`: 连减
  - `mixed_operation`: 加减混合

### 生成器选择指南

| 题目类型 | 推荐生成器 | 配置要点 |
|----------|------------|----------|

| 加减混合 | `mixed_addition_subtraction` | `question_complexity: ["mixed_operation"]` |

## 注意事项

1. **测试驱动（必须）**：先写测试用例，再修改配置/代码，确保测试通过后再提交
2. **先查后改**：先查询理解记录，再进行更新
3. **复用优先**：优先复用现有生成器，不要急于创建新的
4. **约束验证**：扩展生成器后，务必测试约束是否生效
5. **JSON 格式**：`variables_config` 是 JSON 字符串，使用 `DbHelper.update_json_column` 自动序列化
6. **知识点关联**：确保 `knowledge_point_id` 关联的知识点存在且匹配
7. **多维度考虑**：配置时必须同时考虑 name、description、example、knowledge_point.name 等多个维度，确定数字范围和运算类型
8. **result_within 字段**：限制结果上限时用 `result_within`（不是 `result_limit`），如 `"result_within": 5` 表示结果不超过 5
9. **比大小（带运算）题型**：`compare_with_result` 在 `result_within_10` 规则下会生成单步运算（如 `2+4○7`），不是三步混合运算

**常见错误及修复**：
| 错误 | 原因 | 修复 |
|------|------|------|
| 5 以内数生成 5+5=10 | 缺少结果限制 | 添加 `"result_within": 5` |
| 6~10 认识生成 100 以内 | 数字范围设置过大 | 根据知识点调整 `num` 范围和 `result_within` |
| 比大小生成三步运算 | 生成器逻辑问题 | 修复 `compare_with_result` 题型，根据 `result_within_10` 判断生成单步还是多步 |
| 减法出现负数 | 缺少 `ensure_positive` 规则 | 添加 `"rules": ["ensure_positive"]` |
| 比大小出现相等数字 | 缺少 `ensure_different` 规则 | 添加 `"rules": ["ensure_different"]` |

**测试驱动的好处**：
- 避免回归错误：修改后运行测试即可发现是否破坏了已有功能
- 文档化需求：测试用例本身就是需求的可执行说明
- 增强信心：测试覆盖充分时，重构和优化更有底气

## 重要：UTF-8 编码问题解决方案

**问题**：在 Windows 控制台执行 Python 脚本时，中文可能显示为乱码。

**解决方法**：在脚本开头添加以下代码：

```python
# -*- coding: utf-8 -*-
import io
import sys
# 重定向 stdout 为 UTF-8 编码

```

# -*- coding: utf-8 -*-
"""模板配置脚本"""
