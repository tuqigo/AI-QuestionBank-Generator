## 一、项目概述

### 1.1 项目简介

**好学生 AI 题库生成器** 是一款面向学生、家长、老师的 Web 应用，通过 AI 技术自动生成数学、语文等学科的练习题。支持文字提示词生成和图片识别扩展两种模式。

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 用户认证 | 邮箱注册/登录，OTP 验证码，密码找回 |
| 题目生成 | 根据文字提示词 AI 生成题目 |
| 图片扩展 | 上传题目图片，AI 识别后生成同类扩展题 |
| 历史记录 | 保存、查看、删除、分享题目记录 |
| 打印导出 | 支持直接打印或另存为 PDF |
| 管理后台 | 用户管理、禁用控制、操作日志审计 |

## 五、数据库设计

### 5.1 数据库文件

位置：`backend/data/users.db`

### 5.2 表结构

#### users - 用户表
```sql
CREATE TABLE users (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    email           TEXT UNIQUE NOT NULL,
    hashed_password TEXT NOT NULL,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_disabled     INTEGER DEFAULT 0
);
```

#### otp_codes - OTP 验证码表
```sql
CREATE TABLE otp_codes (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    email       TEXT NOT NULL,
    code        TEXT NOT NULL,
    purpose     TEXT NOT NULL DEFAULT 'register',  -- 'register' | 'reset_password'
    attempts    INTEGER DEFAULT 0,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at  TIMESTAMP NOT NULL
);
```

#### otp_rate_limit - OTP 速率限制表
```sql
CREATE TABLE otp_rate_limit (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    email        TEXT NOT NULL,
    purpose      TEXT NOT NULL DEFAULT 'register',
    ip_address   TEXT,
    request_count INTEGER DEFAULT 1,
    reset_at     TIMESTAMP NOT NULL
);
```

#### user_question_records - 用户题目记录表
```sql
CREATE TABLE user_question_records (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    title           VARCHAR(200) NOT NULL,
    prompt_type     VARCHAR(10) NOT NULL,     -- 'text' | 'image'
    prompt_content  TEXT NOT NULL,
    image_path      VARCHAR(500),
    ai_response     TEXT NOT NULL,
    short_id        TEXT UNIQUE,              -- 对外暴露的短 ID（12 字符）
    share_token     VARCHAR(64) UNIQUE,       -- 分享链接 token
    is_deleted      INTEGER DEFAULT 0,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_user_question_records_user_deleted
    ON user_question_records(user_id, is_deleted, created_at DESC);
CREATE INDEX idx_user_question_records_share_token
    ON user_question_records(share_token);
CREATE INDEX idx_user_question_records_short_id
    ON user_question_records(short_id);
```

**short_id 说明**：
- 使用 12 字符随机字符串（A-Za-z0-9），对外暴露替代真实 ID
- 防止遍历攻击和数据泄露（无法通过 ID 推断记录数量）
- 通过 `backend/scripts/migrate_add_short_id.py` 和 `generate_short_ids.py` 迁移历史数据

#### admin_operation_logs - 管理员操作日志表
```sql
CREATE TABLE admin_operation_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    operator    TEXT NOT NULL,
    action      TEXT NOT NULL,
    target_type TEXT,
    target_id   INTEGER,
    ip          TEXT,
    details     TEXT,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_admin_logs_action
    ON admin_operation_logs(action, created_at DESC);
CREATE INDEX idx_admin_logs_target
    ON admin_operation_logs(target_type, target_id);
```

### 9.1 MathJax 公式渲染

前端使用 MathJax 3 SVG 模式渲染 LaTeX 公式：

```typescript
// 加载 MathJax
const loadMathJax = (): Promise<void> => {
  return new Promise((resolve, reject) => {
    if (window.MathJax) {
      resolve()
      return
```

### 9.5 Short ID 机制

**设计目的**：防止通过自增 ID 遍历用户数据，保护隐私

```python
# utils/short_id.py
import secrets
import string

ALPHABET = string.ascii_letters + string.digits  # A-Za-z0-9
SHORT_ID_LENGTH = 12

def generate_short_id() -> str:
    """生成 12 字符随机 ID，例如：nX7kP9mQ2aB3"""
    return ''.join(secrets.choice(ALPHABET) for _ in range(SHORT_ID_LENGTH))
```

**特点**：
- 使用 Python 标准库 `secrets`，无需额外依赖
- 完全兼容 Python 3.8+
- 12 字符长度，碰撞概率极低（约 1/10^21）
- URL 安全，无需编码

**数据迁移**：
```bash
# 1. 添加 short_id 列和索引
python scripts/migrate_add_short_id.py

# 2. 为历史数据生成 short_id
python scripts/generate_short_ids.py
```

## 十、安全机制

### 10.1 密码安全
- 使用 bcrypt 进行密码哈希
- bcrypt 限制密码最大 72 字节（需要截断处理）
- 密码长度验证：6-32 字符

### 10.2 JWT 认证
- 用户 Token 有效期：7 天
- 管理员 Token 有效期：2 小时
- 使用 HS256 算法签名

### 10.3 OTP 安全
- 6 位数字验证码
- 5 分钟有效期
- 最大 5 次尝试
- 速率限制：60 分钟内最多 5 次发送

### 10.4 CORS 配置
```python
# main.py
allow_origins = os.getenv("ALLOW_ORIGINS", "http://localhost:5173").split(",")
app.add_middleware(CORSMiddleware, allow_origins=allow_origins, ...)
```

### 10.5 Short ID 防遍历

**问题**：使用数据库自增 ID 会导致：
1. 可通过遍历 ID 访问他人数据
2. 暴露业务量（如 ID=1000 说明有 1000 条记录）

**解决方案**：
- 对外接口统一使用 12 字符随机 `short_id`
- 数据库内部仍使用自增 `id` 关联
- 所有历史记录相关接口参数改为 `short_id`

**URL 对比**：
| 场景 | 改造前 | 改造后 |
|------|--------|--------|
| 历史详情 | `/history/30` | `/history/nX7kP9mQ2aB3` |
| 分享链接 | `/share/h/30?token=xxx` | `/share/h/nX7kP9mQ2aB3?token=xxx` |
