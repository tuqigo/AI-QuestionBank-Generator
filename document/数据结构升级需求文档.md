## 项目概述

### 项目目标
对系统进行重构，让 AI 输出具有严谨结构的数据，便于前端展示和打印。前端使用固定代码按照题目类型进行渲染，后端按照既定逻辑清洗数据、校验 AI 数据、添加必要字段（如预留行数）。

### 核心原则
1. **新旧隔离**：新增代码不能影响老代码和老逻辑
2. **完整可替换**：新功能开发完成后可以直接替换旧逻辑
3. **不兼容老数据**：老数据不做兼容处理

### 需求来源
- `document/数据结构升级需求` - 原始需求文件
- `document/ai_resp.txt` - AI 输出结果示例

## JSON Schema 设计

### Schema 文件位置
`backend/schemas/question_schema.json`

### 支持的题型 (11 种)

| 题型 | 英文枚举 | 必填字段 | 说明 |
|------|----------|----------|------|
| 单选题 | `SINGLE_CHOICE` | stem, options (4个), knowledge_points | 四选一 |
| 多选题 | `MULTIPLE_CHOICE` | stem, options (≥2个), knowledge_points | 多选 |
| 判断题 | `TRUE_FALSE` | stem, knowledge_points | 正确/错误 |
| 填空题 | `FILL_BLANK` | stem, knowledge_points | 使用 ___ 或 [  ] |
| 计算题 | `CALCULATION` | stem, knowledge_points | 数学计算/解方程 |
| 应用题 | `WORD_PROBLEM` | stem, knowledge_points | 实际场景 |
| 几何题 | `GEOMETRY` | stem, knowledge_points | 涉及图形 |
| 阅读理解 | `READ_COMP` | stem, passage, sub_questions, knowledge_points | 带阅读材料 |
| 古诗文鉴赏 | `POETRY_APP` | stem, knowledge_points | 默写/鉴赏 |
| 完形填空 | `CLOZE` | stem, passage, sub_questions, knowledge_points | 英语完形 |
| 作文题 | `ESSAY` | stem, knowledge_points | 语文/英语作文 |

### Schema 核心结构

```json
{
  "meta": {
    "subject": "math|chinese|english",
    "grade": "grade1~grade9",
    "title": "标题"
  },
  "questions": [
    {
      "type": "题型枚举",
      "stem": "题干",
      "knowledge_points": ["知识点"]
    }
  ]
}
```

## 实现逻辑

### 后端逻辑

#### 1. JSON 校验逻辑
```
AI 返回 JSON → 解析为对象 → 使用 jsonschema 校验 →
├─ 校验通过 → 填充元数据 → 返回
└─ 校验失败 → 返回错误 → (可选) 打给 AI 重新生成
```

#### 2. 元数据填充
后端在返回前需要填充的字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| `rows_to_answer` | int | 预留作答行数（根据题型计算） |
| `difficulty` | string | 难度等级（可选） |
| `created_at` | datetime | 创建时间 |

**预留行数规则（初始版）：**
- `SINGLE_CHOICE` / `TRUE_FALSE` / `FILL_BLANK`：1 行
- `CALCULATION` / `WORD_PROBLEM` / `GEOMETRY`：3 行
- `READ_COMP` / `CLOZE`：题干行数 + 子题数 × 2
- `POETRY_APP` / `ESSAY`：5 行

#### 3. 错误处理
```
校验失败 → 返回 {code: 400, message: "JSON 校验失败", errors: [...]}
```

### 前端逻辑

#### QuestionRenderer 组件逻辑

```typescript
interface QuestionRendererProps {
  question: Question;  // 从 API 返回的结构化数据
}

function QuestionRenderer({ question }: Props) {
  switch (question.type) {
    case 'SINGLE_CHOICE': return <SingleChoice question={question} />;
    case 'MULTIPLE_CHOICE': return <MultipleChoice question={question} />;
    case 'TRUE_FALSE': return <TrueFalse question={question} />;
    case 'FILL_BLANK': return <FillBlank question={question} />;
    case 'CALCULATION': return <Calculation question={question} />;
    case 'WORD_PROBLEM': return <WordProblem question={question} />;
    case 'GEOMETRY': return <Geometry question={question} />;
    case 'READ_COMP': return <ReadComp question={question} />;
    case 'POETRY_APP': return <PoetryApp question={question} />;
    case 'CLOZE': return <Cloze question={question} />;
    case 'ESSAY': return <Essay question={question} />;
    default: return <div>未知题型</div>;
  }
}
```

## AI 提示词调整

需要修改 `backend/config.py` 中的 `QUESTION_SYSTEM_PROMPT`，确保 AI 输出符合 JSON Schema：

1. 确保 AI 只返回 JSON，不包含任何 markdown 标记、解释文字、emoji
2. 所有字段严格按照 Schema 定义的类型和格式输出
3. 答案不要包含在输出中（前端不需要）

### 推荐的 System Prompt 关键点

```
【输出要求】
- 只输出完整、可直接解析的 JSON
- 不要包含 markdown 标记、解释文字、emoji
- 答案不要输出（不需要"## 答案"部分）
- 所有字段严格按 Schema 规定的类型输出
- 题目数量未指定则生成15道
```

### A. JSON Schema 验证工具

```python
# backend/utils/json_validator.py
import jsonschema
from jsonschema import validate, ValidationError

def validate_question_structure(data: dict) -> tuple[bool, list[str]]:
    """验证题目结构"""
    try:
        validate(instance=data, schema=QUESTION_SCHEMA)
        return True, []
    except ValidationError as e:
        return False, [e.message]
```

### B. 预留行数计算参考

```python
def calculate_rows_to_answer(question: dict) -> int:
    type_rows = {
        'SINGLE_CHOICE': 1,
        'TRUE_FALSE': 1,
        'FILL_BLANK': 1,
        'CALCULATION': 3,
        'WORD_PROBLEM': 3,
        'GEOMETRY': 3,
        'POETRY_APP': 5,
        'ESSAY': 5,
    }

    base = type_rows.get(question['type'], 1)

    # 阅读理解/完形填空：题干行数 + 子题数 × 2
    if question['type'] in ['READ_COMP', 'CLOZE']:
        passage_lines = len(question.get('passage', '').split('\n'))
        sub_questions_count = len(question.get('sub_questions', []))
        return passage_lines + sub_questions_count * 2

    return base
```
