## 一、当前问题分析

### 1.1 现状

当前项目首页（根路径 `/`）逻辑：
- 未登录用户 → 自动跳转到 `/login` 登录页
- 已登录用户 → 进入主界面（题目生成/预览）

### 1.2 存在的问题

| 问题 | 说明 | 影响 |
|------|------|------|
| **SEO 几乎为零** | 没有可供搜索引擎索引的内容 | 无法通过搜索获取自然流量 |
| **价值主张不清晰** | 用户不知道产品能做什么 | 跳出率高 |
| **缺乏信任感** | 没有展示、没有案例、没有说明 | 转化率极低 |
| **用户停留时间短** | 登录门槛太高，没有浏览动机 | 流失潜在用户 |

### 1.3 核心结论

**需要一个公开的、SEO 友好的首页（Landing Page）**，让未登录用户也能：
1. 了解产品功能
2. 看到使用价值
3. 建立信任感
4. 自愿注册转化

### 7.2 路由修改

```tsx
// App.tsx
<Routes>
  {/* 管理后台 */}
  <Route path="/admin/*" element={<AdminApp />} />

  {/* 公开分享页面 - 无需登录 */}
  <Route path="/share/h/:id" element={<SharePage />} />

  {/* 【新增】首页 Landing Page - 公开访问 */}
  <Route path="/" element={<LandingPage />} />

  {/* 需要登录的页面 */}
  <Route path="/dashboard" element={user ? <MainContent email={user} onLogout={handleLogout} fetchUser={fetchUser} /> : <Navigate to="/login" />} />
  <Route path="/login" element={user ? <Navigate to="/" /> : <LoginPage onSuccess={fetchUser} />} />
  <Route path="/history/:id" element={user ? <HistoryDetail /> : <Navigate to="/login" />} />
</Routes>
```

**注意**：
- 原根路径 `/` 直接展示主内容 → 改为 `/dashboard`
- 登录页 `/login` 保留，但从首页跳转而非强制重定向
