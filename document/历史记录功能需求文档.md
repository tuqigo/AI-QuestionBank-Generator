# 用户历史记录功能需求文档

## 1. 需求概述

为 AI 题库生成器添加用户历史记录功能，保存用户每次生成题目的记录，支持查看、删除、分享等功能。

### 核心功能
- 用户生成的每条题目记录自动保存
- 每用户最多 1000 条记录（可配置），超出后 FIFO（先进先出）删除
- 支持查看历史记录列表（游标分页）
- 支持查看单条记录详情
- 支持软删除记录
- 支持生成分享链接（带 token 验证）
- 公开分享页面（无需登录即可查看）

## 4. 数据库设计

### 4.1 表结构

```sql
CREATE TABLE user_question_records (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,                     -- 用户 ID
    title           VARCHAR(200) NOT NULL,                -- AI 生成的标题
    prompt_type     VARCHAR(10) NOT NULL,                 -- 'text' 或 'image'
    prompt_content  TEXT NOT NULL,                        -- 文字存全文，图片暂时留白
    image_path      VARCHAR(500),                         -- 预留 S3 URL
    ai_response     TEXT NOT NULL,                        -- AI 返回的题目内容
    share_token     VARCHAR(64) UNIQUE,                   -- 分享 token（主动分享时生成）
    is_deleted      INTEGER DEFAULT 0,                    -- 软删除标记
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP   -- 创建时间
);

-- 索引
CREATE INDEX idx_user_question_records_user_deleted
ON user_question_records(user_id, is_deleted, created_at DESC);

CREATE INDEX idx_user_question_records_share_token
ON user_question_records(share_token);
```

### 4.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | 是 | 自增主键 |
| user_id | INTEGER | 是 | 关联用户表 ID |
| title | VARCHAR(200) | 是 | AI 生成的标题（如"小学一年级 100 以内加减法"） |
| prompt_type | VARCHAR(10) | 是 | 提示词类型：text/image |
| prompt_content | TEXT | 是 | 用户原始提示词（文字存全文，图片暂时留白） |
| image_path | VARCHAR(500) | 否 | 预留字段，后续存 S3 图片 URL |
| ai_response | TEXT | 是 | AI 返回的完整题目内容（Markdown 格式） |
| share_token | VARCHAR(64) | 否 | 分享链接 token，主动分享时生成 |
| is_deleted | INTEGER | 是 | 软删除标记：0-未删除，1-已删除 |
| created_at | TIMESTAMP | 是 | 记录创建时间 |

## 6. 实现思路

### 6.1 题目生成并自动保存流程

```
用户提交 prompt
    ↓
调用 AI 生成 (qwen_client.py)
    ↓
AI 返回 "TITLE: 标题\n\n题目内容..."
    ↓
解析标题和内容 (_parse_title_and_content)
    ↓
保存到数据库 (question_record_store.create_record)
    ↓
返回给用户 (title + markdown)
```

### 6.2 游标分页实现

```python
# 查询 cursor 之前的记录（ID 递减）
SELECT * FROM user_question_records
WHERE user_id = ? AND is_deleted = 0 AND id < ?
ORDER BY id DESC
LIMIT ?
```

- 使用 `id` 作为游标（简单高效）
- `ORDER BY id DESC` 确保最新记录在前
- 多查一条判断 `has_more`

### 6.3 分享链接机制

1. 用户点击"分享" → 生成随机 token（`secrets.token_urlsafe(32)`）
2. token 存入数据库 `share_token` 字段
3. 返回分享链接 `/share/h/:id?token=xxx`
4. 访问分享页 → 通过 token 查询记录（无需登录）
5. 删除记录后 → 分享链接失效（token 关联的记录被软删除）

### 6.4 FIFO 记录限制

```python
current_count = get_user_record_count(user_id)
if current_count >= MAX_RECORDS_PER_USER:
    delete_oldest_record(user_id)  # 软删除最早的一条
```

### 6.5 AI 标题生成

在 qwen_client.py 的 prompt 中添加要求：
```
16. **重要**：在输出内容的最前面，用一行单独的文本输出标题，
    格式为：`TITLE: 你的标题`（不超过 20 字）
```

解析函数：
```python
def _parse_title_and_content(content: str) -> tuple[str, str]:
    match = re.match(r'^TITLE:\s*(.+)$', title_line, re.IGNORECASE)
    if match:
        return match.group(1).strip(), remaining_content
    return "AI 题目生成", content  # 默认标题
```

## 7. 可改进优化方案

### 7.1 性能优化

| 问题 | 优化方案 | 优先级 |
|------|---------|--------|
| SQLite 并发性能 | 迁移到 PostgreSQL/MySQL | 中 |
| 大文本存储 | ai_response 字段压缩存储 | 低 |
| 分享链接查询 | 添加 share_token 缓存（Redis） | 低 |
| 列表加载 | 虚拟滚动（前端） | 中 |

### 7.2 功能增强

| 功能 | 实现方案 | 优先级 |
|------|---------|--------|
| 搜索历史记录 | 添加全文索引/关键词搜索 | 中 |
| 回收站功能 | 添加 deleted_at 字段，定期清理 | 低 |
| 批量删除 | 前端多选 + 后端批量接口 | 低 |
| 记录分类/标签 | 添加 tags 字段 | 低 |
| 导出功能 | 导出 PDF/Word 格式 | 中 |

### 7.3 安全加固

| 风险 | 防护方案 | 优先级 |
|------|---------|--------|
| 分享链接遍历 | token 使用加密哈希（已实现） | 高 |
| 接口滥用 | 添加速率限制（Rate Limiting） | 中 |
| XSS 攻击 | 前端 markdown 渲染时过滤 HTML | 高 |

### 7.4 图片存储优化
